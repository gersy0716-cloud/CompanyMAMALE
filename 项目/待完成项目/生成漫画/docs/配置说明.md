# AI 漫画生成器 — 部署与数据字典

本项目采用纯前端单页架构，去除了多余的业务层参数绑定，以全量硬编码的形式在 `main.js` 中声明了应用所需的接口鉴权、路由配置与数据库结构。

以下为 **全局环境常量表** 与 **BaseMulti 数据库字典**（需参照配置表修改 `main.js` 或在云端建立对应的数据库结构）。

---

## 1. 核心认证与环境常量 (配置于 `main.js`)

### 身份凭证 (Token)

本令牌为全局复用的唯一身份凭证，用于鉴权请求「生图大模型接口 (Mamale.vip)」与「元数据云数据库接口 (BaseMulti)」。在 `main.js` 中需提供此凭证。

| 变量名 | 类型 | 取值说明 / 填入值 | 描述 |
| :--- | :--- | :--- | :--- |
| `AUTH_TOKEN` | SingleLineText | `eyJhbGci...` (使用你的真实 JWT) | 码码乐/BaseMulti 通用鉴权令牌，系统自带剥离业务用户信息。<br>如何获取: 登录 `mamale.vip`，从 `localStorage` 获取 Token 或从请求头中抓取。 |

### 平台基础 API (`config`)

控制 DeepSeek (分镜构思) 与 即梦 AI/Chatgptten (图片生成)核心接口的底层请求逻辑。

| 变量名 | 类型 | 取值说明 / 填入值 | 描述 |
| :--- | :--- | :--- | :--- |
| `apiBase` | URL | `https://3w-api.mamale.vip/api/app` | 码码乐大模型网关入口 |
| `tenant` | SingleLineText | `c1863285-25d1-44fe-805c-5ddf611f83d3` | AI 调用通道租户隔离 ID。将自动注入到 URL 的 `?__tenant=` 参数以避开跨域。 |

## 2. 数据库字典与记录层 (`dbConfig`)

系统依赖 BaseMulti 环境进行漫画预生成的临时保存。在项目上线前，需在 BaseMulti 控制台 (`data.520ai.cc`) 按下表创建对应结构。

### 网关配置与基座 ID

| 变量名 | 类型 | 取值说明 / 填入值 | 描述 |
| :--- | :--- | :--- | :--- |
| `apiBase` | URL | `https://data.520ai.cc/api/bases` | BaseMulti 数据层接入网关 |
| `baseId` | SingleLineText | `bsekddalnVrgIAiZYmM` | 所属项目数据的 Base 数据库系统 ID |
| `tableId` | SingleLineText | `gaHY0ruUUs` | 承载漫画生成逻辑的表系统 ID |

### 数据表结构：AI 漫画 (`gaHY0ruUUs`)

> **[系统自带字段]**：`id` (自增), `name` (记录名), `created_at` (创建时间) 均由系统托管，严禁手动通过 API 传入。

你需要手动创建如下扩展业务字段：

| 字段名 | 字段类型 | 是否必填 | 枚举/映射说明 | 开发侧释义与示例 |
| :--- | :--- | :--- | :--- | :--- |
| `tenantid` | SingleLineText | **必填** | `c1863285-25d1-...` | 租户层级隔离映射字。 |
| `prompt` | LongText | — | — | 原始用户输入故事描述 (`宇航员在火星发现猫`)。 |
| `style` | SingleSelect | — | `[-]` 须与代码的 `PRESETS` key 保持绝对一致，如 `japanese_manga`, `random` 等 | 锁定的绘图预设风格 ID。 |
| `panels_json` | LongText | — | — | 以 JSON 格式缓存返回的四格分镜图元数据。 |
| `panel_images` | LongText | — | — | 以 JSON 数组格式 (`["url1", "url2"]`) 缓存绘画结果。 |
| `status` | SingleSelect | — | `生成中` / `已完成` / `失败` | 追踪整个生产周期的异步生成状态。 |

## 3. 本地跨域规避方案 (CORS)

直接双击 `index.html` (本地 `file:///`) 发送 Fetch 请求至公网 API 会触发防爬防跨域安全机制（报 `401 Unauthorized` 或 CORS 预检拦截）。

必须拉起一个轻量级代理服务，代理本地 `http` 源站：

```bash
# 在含有 index.html 和 main.js 的项目根目录运行
npx -y serve ./ -l 3000
```

然后在浏览器访问: `http://localhost:3000` 即可享受顺畅的测试。
