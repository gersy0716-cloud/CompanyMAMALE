# 班级背诵表 - 数据库表设计 (DATABASE_DESIGN)

本设计基于 **BaseMulti (BM)** 数据库规范。所有字段名推荐使用小驼峰或下划线命名，本项目统一使用 **下划线命名法**。

---

## 1. 核心表清单

### 1.1 recitation_tasks (背诵任务表)

存储教师创建的班级任务及学生创建的个人任务。

| 字段名 | 类型 | 说明 | 备注 |
| :--- | :--- | :--- | :--- |
| **id** | ID | 唯一标识 | 系统生成 |
| **task_name** | Text | 任务名称 | 如《静夜思》 |
| **content** | LongText | 课文原文内容 | 识别/输入后的完整文本 |
| **category** | Select | 课文类别 | 古诗词 / 语文课文 / 英语课文 |
| **passing_score** | Number | 及格得分标准 | 默认 80 |
| **creator_id** | Text | 创建者 ID | 教师 ID 或 学生 ID |
| **creator_type** | Select | 创建者类型 | `teacher` / `student` |
| **class_id** | Text | 班级 ID | 班级任务必填，个人任务可为空 |
| **created_at** | DateTime | 创建时间 | |

### 1.2 recitation_records (背诵记录表)

记录每一次 AI 练习的详细结果及最终的审核状态。

| 字段名 | 类型 | 说明 | 备注 |
| :--- | :--- | :--- | :--- |
| **id** | ID | 唯一标识 | 系统生成 |
| **task_id** | Link | 关联任务 | 关联 `recitation_tasks.id` |
| **student_id** | Text | 学生 ID | 来自 `mamale-api` |
| **student_name**| Text | 学生姓名 | 冗余存储方便查询 |
| **recognized_text**| LongText | 识别后的文本 | 本项目不存音频，仅存文本 |
| **score** | Number | AI 评分分值 | 0-100 |
| **ai_feedback** | Text | AI 评审详情 | DeepSeek 返回的建议 |
| **status** | Select | 状态 | `practice` (练习) / `passed` (已通过) |
| **is_official** | Boolean | 是否为官方认定 | 老师点击通过后置为 true |
| **created_at** | DateTime | 记录时间 | |

### 1.3 students_cache (学生信息缓存)

*可选：若 API 响应较慢，可定时同步缓存。*

| 字段名 | 类型 | 说明 | 备注 |
| :--- | :--- | :--- | :--- |
| **id** | Text | 学生 ID | 业务 ID |
| **name** | Text | 姓名 | |
| **class_id** | Text | 所属班级 ID | |
| **last_sync** | DateTime | 同步时间 | |

---

## 2. 字段映射与约束

### 2.1 状态流转规则

- 学生练习后，`status` 默认为 `practice`。
- 老师在统计表点击“标记通过”或“一键全拔通过”，系统生成/更新一条记录，`status` 设置为 `passed` 且 `is_official` 为 `true`。

### 2.2 索引建议

- `task_id` + `student_id`: 用于快速查询某个学生在特定任务下的最高分或历史评分。
- `class_id`: 教师端展示班级进度矩阵的核心查询字段。
