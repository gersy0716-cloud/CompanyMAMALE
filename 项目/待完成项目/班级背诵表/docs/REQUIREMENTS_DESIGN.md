# 班级背诵表 - 需求与设计详图 (REQUIREMENTS_DESIGN)

## 1. 项目背景与目标

本项目旨在创建一个基于 AI 的班级背诵管理系统。系统通过语音识别和 AI 评分技术，辅助学生进行课文背诵，并为教师提供自动化的管理与统计工具。

---

## 2. 角色与权限 (Roles & Permissions)

系统通过业务 Token 进行身份识别：

- **教师模式** (`?teachertoken=...`): 拥有全部权限，包括创建任务、**设置合格分数阈值**、标记正式完成、**批量通过/全班通过**、**导出报表**。
- **学生模式** (普通 Token): 自动识别当前学生身份，查看个人任务、创建个人任务、进行练习。

---

## 3. 应用场景 (Usage Scenarios)

### 3.1 线下当面背诵 (Face-to-Face Check)

- **场景**: 学生在老师面前背诵，老师开启网页实时监控识别结果。
- **价值**: 辅助老师捕捉漏字、多字，提供客观分值参考，减少人工判断主观性。

### 3.2 自主试背/预习 (Self-Practice)

- **场景**: 学生在早读或课后自主练习。
- **价值**: AI 即时纠错，帮助学生在正式验收前达到熟练状态。

### 3.3 学习成长轨迹 (Growth Tracking)

- **场景**: 系统保留所有历史背诵尝试记录。
- **价值**: 通过数据分析学生对特定课文的熟练过程，识别学习难点。

---

## 4. 核心功能模块 (Core Modules)

### 4.1 教师端 (Teacher Side)

- **任务管理中心**:
  - **智能导入**: 通过 OCR (合合信息) 识别 PDF/图片/Word 课文内容。
  - **规则配置**: 创建任务时设置 **合格得分标准** (如 80分)。
- **背诵统计表**:
  - **进度矩阵**: 纵轴学生，横轴任务，颜色标记完成状态。
  - **批量操作**: 提供“全班通过”选项，或选择部分学生批量标记。
  - **数据导出**: 支持将班级背诵统计表导出为 **Excel 文件**。
  - **记录审核**: 查看学生 AI 练习的历史评分记录。

### 4.2 学生端 (Student Side)

- **背诵看板**: 实时同步班级任务，显示个人最高分与通过状态。
- **AI 练习室**:
  - **实时纠错**: 边背边识别，错词即时标红。
  - **不存音频**: **本项目不存储原始录音文件**，仅保留识别后的文本。
  - **AI 综合评分**: 完成后由 DeepSeek 提供多维度评估。
  - **历史追溯**: **保留背诵历史评分**，方便学生查看进步曲线。
- **个人题库**: 支持学生上传课文进行练习，可自行标记完成。

---

## 5. 技术方案 (Technical Stack)

| 维度 | 技术选型 | 说明 |
| :--- | :--- | :--- |
| **持久化** | **BaseMulti DB** | 码码乐通用数据库 API |
| **OCR** | **合合信息** | 课文内容高精度识别 |
| **语音识别** | **火山豆包** | 500ms 极速流式识别方案 |
| **AI 评分** | **DeepSeek-V3** | 深度语义分析与评分建议 |
| **工具库** | **SheetJS** | 用于 Excel 导出 |
| **UI 视觉** | **天空渐变系** | 现代简约风格，玻璃拟态组件 |

---

## 6. UI/UX 视觉规范 (UI Design)

- **主题**: 天空渐变 (`linear-gradient(135deg, #7DA0CA 0%, #C1E8FF 100%)`)。
- **布局**: Bento 布局，重点页面支持 **拖拽交互** (如任务排序或学生筛选)。
- **动画**: 进入/切换时的平滑淡入效果。

---

## 7. 特殊注意事项

- **Browser-use**: 开发/测试自动化脚本时，**必须**使用 `--browser real` 标志。
