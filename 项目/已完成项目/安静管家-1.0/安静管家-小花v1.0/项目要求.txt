任务：开发一个名为「安静管家」的互动式单页网页应用。

技术栈要求：
*   核心框架：HTML, CSS, JavaScript
*   3D库：Three.js (通过CDN引入，例如 `https://cdnjs.cloudflare.com/ajax/libs/three.js/r128/three.min.js`)
*   UI框架：Tailwind CSS v3 (通过CDN引入，例如 `https://cdn.tailwindcss.com`)
*   图标库：Font Awesome (可选，用于UI图标)
*   代码封装：所有HTML, CSS, JavaScript代码必须整合在**一个单独的HTML文件**中，方便用户下载后直接在浏览器中打开运行。

项目核心需求：

1.  **3D场景与视觉设计：**
    *   **场景设置：** 创建一个充满整个浏览器窗口的WebGL画布。背景色设置为 **深蓝色 #091F2F**。相机初始位置设置为 `(0, 12, 25)`，看向场景中心 `(0, 0, 0)`，提供一个舒适的斜俯视视角。
    *   **土地平面：** 在场景中心创建一个足够大的平面（`PlaneGeometry`）作为土地，例如尺寸为 `40x25`。使用一个带有草地纹理的 `MeshStandardMaterial` 材质。
    *   **光照与阴影：** 添加一个环境光（`AmbientLight`）和一个方向光（`DirectionalLight`）。开启方向光的阴影投射功能，并配置渲染器、土地平面及所有花朵模型以接收和投射阴影，增强真实感。

2.  **小花模型与五阶段生长动画（核心）：**
    *   **模型结构：** 设计一个小花的3D模型，它由多个部分组成：一个花茎（`CylinderGeometry`）、几片叶子（`PlaneGeometry`）和一个花朵（由中心球体`SphereGeometry`和多个花瓣`PlaneGeometry`组成）。
    *   **生长阶段：** 将整个生长过程精确地划分为5个阶段，总时长为60秒。通过修改模型的属性来实现平滑过渡：
        1.  **阶段1 (种子)**: 花茎高度为0，所有部分不可见。
        2.  **阶段2 (发芽)**: 花茎开始生长，顶端长出小叶。参考\小花\1.png
        3.  **阶段3 (成长)**: 花茎继续长高，叶子变大或增多。参考小花\2.png
        4.  **阶段4 (花苞)**: 花茎长到最大高度，顶端形成一个紧闭的花苞。参考小花\3.png
        5.  **阶段5 (盛开)**: 花苞缓缓展开，形成最终形态。参考小花\4.png
    *   **动画驱动：** 创建一个`Flower`类。在类的`update()`方法中，根据一个0到1的`progress`值（代表60秒的生长进度），来动态计算并更新花茎、叶子、花朵各部分的位置、旋转和缩放。

3.  **声音交互与多花种植逻辑（核心）：**
    *   **声音检测：** 使用`navigator.mediaDevices.getUserMedia`获取麦克风权限（需用户交互触发）。使用`AnalyserNode`分析音频数据，计算出实时的音量（分贝值）。
    *   **阈值控制：** 在UI上放置一个HTML滑块，让用户可以设置安静的分贝阈值。
    *   **生长/死亡逻辑：**
        *   **生长中：** 当音量低于阈值时，当前正在培育的花朵`progress`增加。
        *   **盛开：** 当`progress`达到1（即满60秒），这朵花完全盛开。将其保留在场景中，并立即开始培育下一朵。
        *   **死亡：** 当音量高于阈值时，如果场景中存在一个“正在生长”的小花（`progress` < 1），则立即将其从场景中移除。
        *   **重生：** 当音量再次低于阈值时，重新开始培育一朵新花。
    *   **40朵花布局：** 目标是培育出40朵花。采用多行网格布局，例如每行8朵，共5行。新花的X坐标按顺序递增，种满一行后，X坐标重置，Z坐标递减，实现换行。
    *   **数据持久化：** 使用`localStorage`保存**已成功盛开的花朵数量**，刷新页面后能恢复场景中的花朵。

4.  **UI与交互设计：**
    *   **视觉风格：** 整体风格清新自然、现代简约。灵活运用指定的五种颜色：#091F2F, #2C677B, #5FA684, #B0FC84, #E1FCC0。为按钮、卡片等关键元素添加从`#5FA684`到`#B0FC84`的渐变色。
    *   **布局结构：**
        *   **顶部标题栏：** 页面顶部居中，固定定位，半透明背景，显示「安静小花」标题。
        *   **右上角状态指示器：** 一个小圆形图标，安静时为绿色，吵闹时为红色。
        *   **底部控制面板：** 页面底部，固定定位，半透明深色背景（#2C677B/80），圆角设计。
    *   **控制面板组件：**
        *   信息卡片：显示实时分贝、已开花数/40、剩余时间。
        *   分贝阈值滑块。
        *   功能按钮：音效开关、重置进度。
        *   生长进度条。
    *   **最终状态：** 当40朵花全部盛开后，显示一个成功弹窗，并提供“重新开始”按钮。

5.  **代码质量与注释：**
    *   代码结构清晰，使用ES6的`class`来组织代码。
    *   在关键逻辑处（如音量分析、生长/死亡判断、动画更新）添加详细的中文注释。
    *   优雅处理用户拒绝麦克风权限等边界情况。

输出要求：
*   提供一个完整的、独立的HTML文件。
*   文件中需包含所有必要的HTML、CSS（通过Tailwind实现）和JavaScript代码。
*   所有外部库（Three.js, Tailwind CSS）通过CDN链接引入。
*   代码可以直接在现代浏览器（如Chrome, Edge）中打开并运行。
