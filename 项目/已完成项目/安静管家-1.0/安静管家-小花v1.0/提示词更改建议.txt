1.布局多次修改，没有一次给定最终版本
【布局最终要求】
页面底部仅一个控制面板，分左右两个模块组：
左侧模块组（从上到下）：实时分贝、已开花数、剩余时间；
右侧模块组（从上到下）：分贝阈值、生长进度、监听按钮 | 重置进度按钮；
布局规则：
宽度：左侧每个子模块的宽度，强制等于右侧整个模块组的实际渲染宽度（先获取右侧容器宽度，再赋值给左侧 3 个子模块）；
对齐：左右模块组顶部边缘齐平、底部边缘齐平（用 Flex 布局，父容器设 align-items: flex-start，通过 margin/padding 微调底部）；
样式：模块宽度自适应内容，white-space: nowrap，模块间 margin-bottom:15px，不强制拉伸。
2. 补充「技术实现细节」，避免 Claude “猜操作”
（1）重置功能：只说 “清零已开花数、生长进度、剩余时间”，没说 “清除本地存储（localStorage.clear ()）+ 同步更新 UI 显示 + 花朵模型重置为初始状态”；
（2）缩放要求：只说 “缩放只针对导航栏”，没说 “导航栏容器加 transform: scale ()，标题「安静管家」和右上角状态指示器设 position: fixed + transform: none，避免跟随缩放”。
优化建议：
对关键功能，明确「代码操作步骤」，让 Claude 直接执行，无需推测。
示例（重置功能 + 缩放功能）：
【重置进度按钮（强制生效）】
点击后执行以下 3 步操作：
数据清零：已开花数 = 0、生长进度 = 0、剩余时间 = 0；
清除存储：执行 localStorage.clear ()，取消所有本地存档（重开网页效果与点击重置一致）；
UI + 模型同步：更新页面显示的数值为 0，同时将花圃中所有花朵模型销毁 / 隐藏，恢复初始无花状态。
【页面缩放规则】
仅导航栏（底部控制面板）支持缩放，其他元素（标题「安静管家」、右上角状态指示器）不跟随缩放；
代码实现：
导航栏容器添加缩放样式（如 transform: scale (1.1)），不影响父容器布局；
标题和状态指示器设 position: fixed，单独设置 top/left/right 定位，添加 transform: none !important，禁止继承缩放属性。
