# 班级值日表系统 - 项目说明

## 📋 项目概述

班级值日表管理系统，支持教师配置值日安排和学生查看值日表。

**当前版本**: 1.0  
**最后更新**: 2026-01-12

## 🎯 核心功能

### 教师端

1. **班级管理** - 查看和管理名下班级（使用teachertoken自动过滤）
2. **小组管理** - 创建、编辑、删除值日小组
3. **值日配置** - 交互式年度日历选择值日日期
4. **自动排班** - 基于小组和日期自动生成值日表
5. **值日表查看** - 日历形式展示，悬浮显示详情

### 学生端

1. **今日值日** - 显示当日值日安排
2. **本周值日** - 查看一周值日表

## 🎨 技术亮点

### 1. 年度日历视图

- **12个月网格布局** - 4列响应式设计
- **节假日自动获取** - 集成timor.tech API
- **默认工作日选择** - 自动选中周一至周五，排除节假日
- **圆形日期格** - 柔和配色，视觉优雅
- **响应式字体** - `max(10px, min(0.9vw, 14px))`随窗口缩放

### 2. 学年与日历年份分离

- **学年选择器** - 配置归属的学年（如2025-2026学年）
- **日历显示年份** - 独立控制显示哪一年的日历
- **灵活切换** - 可以为2025-2026学年选择2027年的日期

### 3. 值日表可视化

- **日历形式展示** - 替代列表视图
- **悬浮提示** - 鼠标hover显示值日组和成员
- **视觉标记** - 有值日的日期显示小圆点

### 4. 性能优化

- **批量查询** - 一次性获取所有记录，客户端过滤
- **移除冗余查询** - 不预加载学生人数
- **并行请求** - Promise.all提升加载速度

## 🗄️ 数据库表设计

### 1. duty_groups (值日小组)

- `id` - 主键
- `class_id` - 班级ID
- `name` - 小组名称
- `student_ids` - 学生名单(JSON)
- `order_index` - 排序序号

### 2. duty_schedule_config (值日配置)

- `id` - 主键
- `class_id` - 班级ID
- `year` - 学年
- `selected_dates` - 选中的日期(JSON)
- `start_date` - 起始日期
- `end_date` - 结束日期
- `is_active` - 是否启用(Switch字段，值为1/0)

### 3. duty_assignments (值日分配)

- `id` - 主键
- `class_id` - 班级ID
- `duty_date` - 值日日期
- `group_id` - 小组ID
- `group_name` - 小组名称(冗余存储)
- `student_ids` - 成员名单(JSON，冗余存储)

## 🔧 重要配置

### API端点

- **BaseMulti数据库**: `https://data.520ai.cc/`
- **节假日API**: `http://timor.tech/api/holiday/year/{year}`
- **班级API**: `https://{type}.mamale.vip/api/app/class`
- **学生API**: `https://{type}.mamale.vip/api/app/student/byClssId`

### 开发服务器

- **教师端**: `npm run dev:teacher` (端口5173)
- **学生端**: `npm run dev:student` (端口5174)

### URL参数

- `type` - 类型标识
- `tenant` - 租户ID
- `userid` - 用户ID
- `username` - 用户名
- `token` - 用户令牌
- `teachertoken` - 教师令牌(教师端必需)

## 💡 关键技术决策

### 1. 日历状态管理

```javascript
const calendarState = {
    selectedYear: 2026,           // 学年选择
    calendarDisplayYear: 2026,    // 日历显示年份
    selectedDates: new Set(),     // 选中的日期
    holidays: new Map(),          // 节假日缓存
    // ...
};
```

### 2. BaseMulti数据库特性

- **Switch字段**: 返回数字1/0，不是布尔值
- **JSON字段**: 必须为字符串格式
- **Filter参数**: 不生效，需客户端过滤
- **ID类型**: 可能是字符串或数字，使用宽松比较

### 3. 日期处理

- **避免UTC转换**: 不使用`toISOString()`
- **本地格式化**: 手动拼接`YYYY-MM-DD`
- **时区处理**: 使用本地日期，避免日期错位
