# 班级值日表系统 1.0

## 📋 项目简介

班级值日表管理系统，支持教师配置值日安排和学生查看值日表，采用温暖米棕配色，提供优雅的用户体验。

**当前版本**: 1.0  
**最后更新**: 2026-01-12 21:40  
**技术栈**: Vanilla JS + Vite + BaseMulti数据库

## 🎨 设计特色

- **温暖米棕配色** - 优雅舒适的视觉体验
- **年度日历视图** - 12个月网格布局，响应式设计
- **玻璃拟态效果** - 重要内容卡片使用半透明玻璃效果
- **渐变色卡片** - 普通列表使用柔和渐变
- **新拟态按钮** - 3D立体效果的操作按钮
- **脉冲动画** - 重要提醒自动呼吸动画
- **节假日集成** - 自动获取国家法定节假日

## 🗄️ 数据库说明

### 数据库平台

- **BaseMulti** - <https://data.520ai.cc/>
- 需要配置API Key和Base ID

### 使用的表

1. **duty_groups** - 值日小组表
   - `id` - 主键
   - `class_id` - 班级ID
   - `name` - 小组名称
   - `student_ids` - 学生名单（JSON格式）
   - `order_index` - 排序序号

2. **duty_schedule_config** - 值日配置表
   - `id` - 主键
   - `class_id` - 班级ID
   - `year` - 学年
   - `selected_dates` - 选中的日期（JSON格式）
   - `start_date` - 起始日期
   - `end_date` - 结束日期
   - `is_active` - 是否启用（Switch字段，值为1/0）

3. **duty_assignments** - 值日分配表
   - `id` - 主键
   - `class_id` - 班级ID
   - `duty_date` - 值日日期
   - `group_id` - 小组ID
   - `group_name` - 小组名称（冗余存储）
   - `student_ids` - 成员名单（JSON格式，冗余存储）

### 表ID配置

在 `database.js` 中配置表ID：

```javascript
const TABLES = {
    duty_groups: 'your_table_id',
    duty_schedule_config: 'your_table_id',
    duty_assignments: 'your_table_id'
};
```

## 📝 使用流程

### 教师端操作流程

1. 使用带teachertoken的URL访问系统
2. 在班级列表中选择要管理的班级
3. 创建值日小组并添加学生成员
4. 进入"值日配置"页面
   - 选择学年（如2025-2026）
   - 在日历上选择需要值日的日期
   - 系统自动排除节假日和周末（可自定义）
5. 保存配置
6. 点击"生成值日表"自动排班
7. 在"值日表"页面查看生成的安排

### 学生端访问

1. 使用不带teachertoken的URL访问系统
2. 系统自动显示本周值日安排
3. 今日值日在顶部玻璃卡片中显示

## 🎯 核心算法

### 自动排班算法

```
1. 获取小组列表（按order_index排序）
2. 获取配置中选定的日期列表
3. 过滤当前有效的学生（排除已删除学生）
4. 循环分配：
   日期1 → 小组1
   日期2 → 小组2
   ...
   日期N → 小组(N % 小组数)
5. 批量创建值日分配记录
```

### 日期选择逻辑

```
1. 加载指定年份的日历
2. 从timor.tech获取国家法定节假日
3. 默认选中所有工作日（周一至周五）
4. 排除节假日和调休日
5. 用户可手动添加/移除特定日期
6. 保存到selected_dates字段
```

## ⚠️ 注意事项

1. **首次使用** - 必须按顺序：创建小组 → 配置日期 → 生成值日表
2. **重新生成** - 会删除旧的值日表数据，请谨慎操作
3. **学生过滤** - 系统会自动过滤已删除的学生，无需手动清理
4. **Token有效期** - Token会过期，需要重新获取
5. **浏览器要求** - 建议使用Chrome、Edge、Safari等现代浏览器
6. **CORS配置** - Vite已启用CORS，无需额外配置

## 🔧 故障排除

### npm未找到

- 下载安装Node.js：<https://nodejs.org/>
- 推荐使用LTS版本

### 端口被占用

编辑 `package.json`，修改对应的端口号：

```json

```

### 依赖安装失败


### API调用失败

- 检查网络连接
- 确认token是否有效
- 查看浏览器控制台错误信息

- 检查database.js中的配置
- 确认API Key和Base ID正确
