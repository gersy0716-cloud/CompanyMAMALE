## 二、API 接口说明

### 2.1 认证方式


### 2.2 获取记录列表 API

#### 接口说明

获取表格中的记录列表，支持分页、排序和筛选。

#### 请求方法

```http
GET /api/bases/{base_id}/tables/{table_name}/records
```

#### 路径参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| `base_id` | string | 数据库 ID |
| `table_name` | string | 表格名称 |

#### 查询参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `page` | number | 1 | 页码 |
| `pageLimit` | number | 15 | 每页记录数（最大100） |
| `filters` | string | - | 筛选条件（base64编码） |

#### 筛选条件格式

筛选条件是将条件数组使用 base64 编码后的字符串。支持的操作符：

| 操作符 | 说明 | 示例 |
|--------|------|------|
| `=` | 等于 | `["where", ["age", "=", 18]]` |
| `!=` | 不等于 | `["where", ["age", "!=", 18]]` |
| `>` | 大于 | `["where", ["age", ">", 18]]` |
| `>=` | 大于等于 | `["where", ["age", ">=", 18]]` |
| `<` | 小于 | `["where", ["age", "<", 18]]` |
| `<=` | 小于等于 | `["where", ["age", "<=", 18]]` |
| `in` | 在列表中 | `["where", ["status", "in", ["active", "pending"]]]` |
| `notin` | 不在列表中 | `["where", ["status", "notin", ["deleted"]]]` |
| `like` | 包含 | `["where", ["name", "like", "test"]]` |
| `notlike` | 不包含 | `["where", ["name", "notlike", "test"]]` |
| `isNull` | 为空 | `["where", ["description", "=", null]]` |
| `isNotNull` | 不为空 | `["where", ["description", "!=", null]]` |

#### 筛选条件示例

**基础查询：**

```javascript
// SELECT * FROM users WHERE age > 18;
const encoded = btoa(JSON.stringify([
  ["where", ["age", ">", 18]]
]));
```

**多条件查询：**

```javascript
// SELECT * FROM users WHERE age > 18 AND status = 'active';
const encoded = btoa(JSON.stringify([
  ["where", ["age", ">", 18]],
  ["where", ["status", "=", "active"]]
]));
```

**排序：**

```javascript
// SELECT * FROM users ORDER BY age DESC;
const encoded = btoa(JSON.stringify([
  ["orderBy", ["age", "desc"]]
]));
```

**字段选择：**

```javascript
// SELECT name, age FROM users;
const encoded = btoa(JSON.stringify([
  ["select", ["name", "age"]]
]));
```

**复合条件：**

```javascript
const encoded = btoa(JSON.stringify([
  ["select", ["id", "title", "description"]],
  ["with", ["author:id,name"]],
  ["where", ["vote", ">", 10]],
  ["orderBy", ["votes", "desc"]]
]));
```

#### 请求示例

```http
# 基础查询
GET /api/bases/base123/tables/table456/records

# 分页查询
GET /api/bases/base123/tables/table456/records?page=2&pageLimit=10

# 筛选查询
GET /api/bases/base123/tables/table456/records?filters=W1sid2hlcmUiLCBbImFnZSIsICI+IiwgMThdXV0=
```

#### 响应格式

```json
{
  "current_page": 1,
  "data": [
    // 记录数组
  ],
  "per_page": 15,
  "total": 10,
  "last_page": 1,
  "count": 10
}
```

### 2.3 获取单条记录 API

#### 接口说明

获取表格中指定记录的详细信息。

#### 请求方法

```http
GET /api/bases/{base_id}/tables/{table_name}/records/{record_id}
```

#### 路径参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| `base_id` | string | 数据库 ID |
| `table_name` | string | 表格名称 |
| `record_id` | string | 记录 ID |

#### 查询参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| `fields` | string | 要返回的字段列表，用逗号分隔 |

#### 请求示例

```http
# 基础查询
GET /api/bases/base123/tables/table456/records/rec789

# 指定字段
GET /api/bases/base123/tables/table456/records/rec789?fields=id,name,created_at
```

### 2.4 创建记录 API

#### 接口说明

在指定表格中创建一条新记录。

#### 请求方法

```http
POST /api/bases/{base_id}/tables/{table_name}/records
```

#### 路径参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| `base_id` | string | 数据库 ID |
| `table_name` | string | 表格名称 |

#### 请求头

```http
x-bm-token: your_api_token
Content-Type: application/json
```

#### 请求体

> [!IMPORTANT]
> **重要提醒**：创建记录时，直接传递字段名和值，**不需要** `fields` 对象包裹！

**基础字段：**

```json
{
  "name": "测试记录",
  "description": "这是一条测试记录",
  "age": 25,
  "is_active": true
}
```

**选项字段：**

```json
{
  "status": "opt123",  // 单选字段使用选项 ID
  "tags": ["opt456", "opt789"]  // 多选字段使用选项 ID 数组
}
```

**关联字段：**

```json
{
  "single_link": "rec123",  // 单关联使用记录 ID
  "multiple_link": ["rec123", "rec456"]  // 多关联使用记录 ID 数组
}
```

#### 响应格式

```json
{
  "id": "11",
  "name": "test",
  "created_by": "usrIL1t20OwVvW9jXzT",
  "updated_by": "usrIL1t20OwVvW9jXzT",
  "created_at": "2024-11-20T17:30:12.000Z",
  "updated_at": "2024-11-20T17:30:27.000Z"
}
```
