## 🚀 构建配置

已在 `vite.config.ts` 中配置资源打包策略：

```typescript
build: {
  // 小于 4KB 的资源内联为 base64
  assetsInlineLimit: 4096,

  // 静态资源分类打包
  rollupOptions: {
    output: {
      chunkFileNames: 'assets/js/[name]-[hash].js',
      entryFileNames: 'assets/js/[name]-[hash].js',
      assetFileNames: 'assets/[ext]/[name]-[hash].[ext]',
    }
  },

  // 代码压缩
  minify: 'terser',
}
```

## 📂 构建产物结构

运行 `npm run build` 后，`dist` 目录结构：

```
dist/
├── index.html                    # 入口 HTML
├── assets/
│   ├── js/
│   │   ├── index-[hash].js      # 主应用 JS
│   │   ├── vendor-[hash].js     # 第三方库
│   │   └── ...
│   ├── css/
│   │   └── index-[hash].css     # 样式文件
│   └── [其他资源]/
```

**所有资源都在 `dist` 目录中，无需外部网络即可运行**（除了必需的 API 调用）。

## 🔒 离线部署说明

### 1. 构建生产版本
```bash
npm run build
```

### 2. 部署到服务器
将 `dist` 目录的内容部署到：
```
/plat/public/20251220005410/
```

### 3. 网络要求
- ✅ 前端资源：**完全本地化**，无需外网
- ⚠️ API 调用：需要访问以下域名
  - `dalu.chatgptten.com`（文本生成）
  - `3w-api.mamale.vip`（图片生成）

如果在内网部署，需要确保服务器能访问上述 API 地址，或部署私有 API 服务。
