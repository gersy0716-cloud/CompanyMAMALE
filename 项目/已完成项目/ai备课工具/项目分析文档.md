# AI备课工具 - 项目分析文档

## 一、项目概述

**项目名称**: AI备课工具 (gemini-ppt-workbench)
**项目类型**: React + TypeScript 单页应用 (SPA)
**主要功能**: 基于AI的智能PPT课件生成工具，专为中小学教师设计

## 四、数据库表结构分析

### 4.1 数据库架构说明

**重要**: 本项目是纯前端应用,不直接操作数据库。所有数据库操作通过后端API完成。

### 4.2 涉及的数据库表 (推断)

根据API接口分析,后端应该包含以下数据表:

#### 表1: aiPPTX (课件表)
```sql
-- 课件主表
CREATE TABLE aiPPTX (
    id VARCHAR(36) PRIMARY KEY,                    -- 课件ID
    name VARCHAR(255) NOT NULL,                    -- 课件名称
    converImg VARCHAR(500),                        -- 封面图URL
    pptData TEXT,                                  -- 课件JSON数据URL
    aiApplicationCategoryId VARCHAR(36),           -- 分类ID (外键)
    isShare BOOLEAN DEFAULT false,                 -- 是否分享
    slideCount INT,                                -- 页数
    viewCount INT DEFAULT 0,                       -- 浏览次数
    creatorId VARCHAR(36),                         -- 创建者ID
    creatorName VARCHAR(100),                      -- 创建者名称
    tenantId VARCHAR(36),                          -- 租户ID (校区)
    creationTime DATETIME DEFAULT CURRENT_TIMESTAMP,
    lastModificationTime DATETIME,
    INDEX idx_category (aiApplicationCategoryId),
    INDEX idx_creator (creatorId),
    INDEX idx_tenant (tenantId)
);
```

#### 表2: aiApplicationCategory (分类表)
```sql
-- 课件分类表 (学科分类)
CREATE TABLE aiApplicationCategory (
    id VARCHAR(36) PRIMARY KEY,
    name VARCHAR(100) NOT NULL,                    -- 分类名称 (如: 语文、数学)
    fullName VARCHAR(255),                         -- 完整名称
    code VARCHAR(50),                              -- 分类代码
    level INT,                                     -- 层级
    sortId INT,                                    -- 排序ID
    parentId VARCHAR(36),                          -- 父分类ID
    logo VARCHAR(500),                             -- 分类图标
    count INT DEFAULT 0,                           -- 课件数量
    creationTime DATETIME DEFAULT CURRENT_TIMESTAMP,
    lastModificationTime DATETIME,
    INDEX idx_parent (parentId)
);
```

#### 表3: aiCourseMethod (课表目录表)
```sql
-- 课表目录树
CREATE TABLE aiCourseMethod (
    id VARCHAR(36) PRIMARY KEY,
    name VARCHAR(100) NOT NULL,                    -- 目录名称
    fullName VARCHAR(255),                         -- 完整路径
    code VARCHAR(50),                              -- 目录代码
    level INT,                                     -- 层级
    sortId INT,                                    -- 排序
    parentId VARCHAR(36),                          -- 父目录ID
    count INT DEFAULT 0,                           -- 包含课件数
    creationTime DATETIME DEFAULT CURRENT_TIMESTAMP,
    lastModificationTime DATETIME,
    INDEX idx_parent (parentId)
);
```

#### 表4: aiCourseMethodApplication (课表应用关联表)
```sql
-- 课件与课表的关联表
CREATE TABLE aiCourseMethodApplication (
    id VARCHAR(36) PRIMARY KEY,
    aiCourseMethodId VARCHAR(36) NOT NULL,         -- 课表目录ID
    aiPPTId VARCHAR(36) NOT NULL,                  -- 课件ID
    targetType INT,                                -- 类型 (3=aiPPTX)
    name VARCHAR(255),                             -- 课件名称
    creationTime DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_method (aiCourseMethodId),
    INDEX idx_ppt (aiPPTId),
    FOREIGN KEY (aiCourseMethodId) REFERENCES aiCourseMethod(id),
    FOREIGN KEY (aiPPTId) REFERENCES aiPPTX(id)
);
```

#### 表5: fileResouceItem (文件资源表)
```sql
-- 文件上传记录表
CREATE TABLE fileResouceItem (
    id VARCHAR(36) PRIMARY KEY,
    fileName VARCHAR(255),                         -- 文件名

    fileType VARCHAR(50),                          -- 文件类型
    uploaderId VARCHAR(36),                        -- 上传者ID
    creationTime DATETIME DEFAULT CURRENT_TIMESTAMP

```

#### 表6: tuZiAsyncTask (异步任务表 - 图片生成)
```sql
-- 图片生成异步任务表
CREATE TABLE tuZiAsyncTask (
    taskId VARCHAR(36) PRIMARY KEY,
    model VARCHAR(100),                            -- 模型名称
    prompt TEXT,                                   -- 提示词
    size VARCHAR(20),                              -- 尺寸 (如: 16:9)
    status VARCHAR(20),                            -- 状态: pending/processing/completed/failed
    imageUrl VARCHAR(500),                         -- 生成的图片URL
    userId VARCHAR(36),                            -- 用户ID
    creationTime DATETIME DEFAULT CURRENT_TIMESTAMP,
    completionTime DATETIME,
    INDEX idx_status (status),
    INDEX idx_user (userId)
);
```

## 六、关键数据流

### 6.1 课件生成流程

```
用户输入文本
    ↓
调用 generateOutline() [geminiService.ts]
    ↓ (使用 OpenAI Stream API)
生成大纲结构 (Slide[])
    ↓
用户编辑大纲
    ↓
批量生成图片 generateSlideImageAsync()
    ↓ (异步任务模式)
1. 创建任务 (asyncImageCreateMy)
2. 轮询查询 (asyncImageQueryMy)
3. 获取图片URL
4. 转换到阿里云 (myConvertUrl)
    ↓
生成语音 (可选) [ttsService.ts]
    ↓
生成视频 (可选) [videoGenerationService.ts]
    ↓
添加互动工具 (可选)
    ↓
导出 PPTX/PDF 或 保存到云端
```

### 6.2 云端保存流程

```
用户点击"保存到云端"
    ↓
选择分类、输入名称、选择是否分享
    ↓
1. 生成JSON数据 (slides + settings)
    ↓
2. 上传JSON到OSS (uploadJSONFile)
    ↓ 返回 pptData URL
3. 生成封面图 (第一页截图)
    ↓
4. 调用 saveToCloud() 或 updateCourseware()
    ↓ POST /app/aiPPTX/my 或 PUT /app/aiPPTX/{id}/my
保存课件元数据到数据库
    ↓
返回课件中心
```

### 6.3 课件加载流程

```
用户点击"查看"或"编辑"
    ↓
调用 getCoursewareDetail(id)
    ↓ GET /app/aiPPTX/{id}/my
获取课件元数据 (包含 pptData URL)
    ↓
fetch(pptData URL)
    ↓
解析JSON数据 (slides + settings)
    ↓
恢复课件状态
    ↓
进入工作台 (WORKBENCH)
```

## 八、核心算法与逻辑

### 8.1 大纲生成算法

```typescript
// geminiService.ts - generateOutline()
1. 构建系统提示词:
   - 根据学科、风格、内容侧重生成提示词
   - 包含35种预设风格的详细描述
   - 包含讲解风格和文案风格

2. 调用OpenAI Stream API:
   - 模型: gemini-3-pro-preview
   - 流式返回JSON数据
   - 实时解析并更新UI

3. 解析返回的JSON:
   - 提取slides数组
   - 每个slide包含: title, content, visualPrompt
   - 如果启用语音,还包含 voiceScript
```

### 8.2 图片生成算法

```typescript
// geminiService.ts - generateSlideImageAsync()
1. 构建视觉提示词:
   - 结合slide.title, slide.content, slide.visualPrompt
   - 添加风格描述
   - 要求16:9宽高比

2. 异步任务模式:
   - 创建任务 (POST asyncImageCreateMy)
   - 获取taskId
   - 轮询查询 (POST asyncImageQueryMy)
   - 最大轮询30次,间隔2秒
   - 超时控制150秒

3. URL转换:
   - 获取图片URL后
   - 调用 convertUrlToAliyun()
   - 转换为阿里云OSS地址
   - 提高访问速度和稳定性
```

### 8.3 语音合成算法

```typescript
// ttsService.ts - generateVoice()
1. 调用火山引擎TTS API:
   - 提交任务 (POST)
   - 获取taskId

2. 轮询查询:
   - 最大轮询30次
   - 间隔2秒
   - 检查status: pending/processing/completed/failed

3. 获取音频URL:
   - 任务完成后返回MP3 URL
   - 存储到slide.voiceUrl
```

### 8.4 视频生成算法

```typescript
// videoGenerationService.ts - generateVideo()
1. 图片预处理:
   - 压缩图片到1024宽度 (imageCompression.ts)
   - 上传压缩后的图片
   - 获取图片URL

2. 创建视频任务:
   - 模型: sora-2
   - 图片转视频 (Image-to-Video)
   - 提供提示词

3. 轮询查询:
   - 最大轮询100次
   - 间隔3秒
   - 预计60-90秒完成

4. 获取视频URL:
   - 存储到slide.videoUrl
```

```typescript
try {
  const response = await fetch(apiUrl, {...});
  if (!response.ok) {
    const errorText = await response.text();
    throw new Error(`API调用失败 (${response.status}): ${errorText}`);
  }
  return await response.json();
} catch (error) {
  console.error('API错误:', error);
  throw new Error('操作失败: ' + (error as Error).message);
}
```

### 12.2 超时控制

- 语音合成失败可重新生成

## 十四、总结

### 14.1 项目特点

1. **纯前端应用**: 不依赖后端服务器,通过API与后端交互
2. **多模态内容**: 支持文本、图片、语音、视频
3. **丰富的风格系统**: 35种预设风格 + 自定义
4. **强大的编辑能力**: 图片编辑、绘图标注、互动工具
5. **云端协作**: 课件中心、分类管理、共享功能

### 14.2 技术亮点

1. **异步任务处理**: 图片/语音/视频生成使用异步任务模式
2. **流式API**: 大纲生成使用SSE流式传输
3. **状态管理**: 使用History API管理应用状态
4. **拖拽排序**: 使用@hello-pangea/dnd实现大纲拖拽
5. **多格式导出**: 支持PPTX、PDF、JSON导出

### 14.3 数据库设计

虽然前端不直接操作数据库,但通过API接口可以推断出后端数据库包含以下核心表:
- aiPPTX (课件表)
- aiApplicationCategory (分类表)
- aiCourseMethod (课表目录表)
- aiCourseMethodApplication (课表应用关联表)
- fileResouceItem (文件资源表)
- tuZiAsyncTask (异步任务表)
