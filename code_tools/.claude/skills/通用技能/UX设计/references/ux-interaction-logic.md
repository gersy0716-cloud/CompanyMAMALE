# 用户交互逻辑偏好

## 加载体验 (Loading Experience)

- **全屏遮罩**: 结合 `backdrop-filter` 模糊效果，提升加载过程质感。
- **按需加载**: 仅在必要处显示 Spinner。空状态容器应优先显示引导。
- **生命周期**: 异步操作开始即显，成功或报错即隐。

## UI 引导与提示 (UI Guidance)

- **显式提示**: 为空状态提供具体指令（如“拖拽学生到这里”）。
- **表单认证**: 优先社交登录；申请敏感权限（如位置）前先用 UI 解释目的。
- **确认机制**: 危险操作（删除、覆盖）必须弹窗确认，并列出受影响范围。

## 页面导航与状态 (Navigation & State)

- **返回重置**: 返回按钮应清空当前流程状态。
- **稳定导航**: 页面切换使用 `switch` 映射固定 ID，通过 `classList` 和 `style.display` 双重同步显示。
- **动态 CTA**: 导航栏操作随登录态动态变化，保证最高价值引导始终可见。

## 状态设计策略

- **标记管理**: 使用 `class-selected` 等专用类名通过 CSS 控制 UI 响应。
- **数据共享**: 核心数据保存在 `APP_STATE` 中，避免复杂的跨模块状态传递。

## 禁用/避坑项

❌ **玻璃徽章** - 避免在小型徽章标签上使用毛玻璃效果，由于识别度低会损害 UX。
