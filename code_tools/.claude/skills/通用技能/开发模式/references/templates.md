# Dev Docs 模板参考

直接复制以下模板用于初始化开发任务文档。

---

## 1. Plan 模板 ([task-name]-plan.md)

```markdown
# [任务名称] - 实施计划

## 🎯 目标摘要
简述任务目标与交付物。

## 🏗️ 实施阶段
### 阶段 1: 基础设施
- [ ] 任务 A (验收标准: ...)
- [ ] 任务 B

### 阶段 2: 核心功能
...
```

---

## 2. Context 模板 ([task-name]-context.md)

> [!IMPORTANT]
> **本文件应在开发过程中保持最新。**

```markdown
# [任务名称] - 开发上下文

## 🕒 SESSION PROGRESS (YYYY-MM-DD)

### ✅ 已完成
- 开发了 X 功能
- 修复了 Y Bug

### 🟡 进行中
- 正在处理 Z 逻辑 (涉及文件: `src/utils/z.ts`)

### ⚠️ 阻塞项 / 待决定
- ...

## 🗝️ 关键决策
- 决策 A: 为什么选择 B 方案而非 C。

## 📖 快速恢复指令
1. 查看 `tasks.md` 确认下一个勾选点。
2. 接着 `z.ts` 的第 50 行继续重构。
```

---

## 3. Tasks 模板 ([task-name]-tasks.md)

```markdown
# [任务名称] - 任务清单

## 🚀 进度总览
- 阶段 1: ✅
- 阶段 2: 🟡
- 阶段 3: ⏳

## 📝 详细检查清单
- [x] 配置数据库
- [ ] 实现 API (进行中)
- [ ] 编写测试
```
