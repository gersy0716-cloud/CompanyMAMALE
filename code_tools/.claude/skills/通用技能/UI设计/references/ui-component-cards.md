# 卡片组件规范

## 信息卡片（单体内容）

承载独立信息单元（值日信息、课程简介）。

**结构**：

- 标题区：名称、日期
- 内容区：关键字段汇总
- 操作区：查看详情、编辑、删除

**交互**：

- 整个卡片可点击进入详情
- 局部操作预留明确点击区域
- 列表中高度和内边距保持一致
- **2x 圆角法则**：当内含元素也有圆角时，外层半径 = 内层半径 * 2，确保视觉平行感。
- **显式 CTA**：不要只依赖悬浮点击，在卡片内提供一个清晰的按钮或文字链接作为视觉锚点。
- **Padding 平衡**：若内部元素较多且不对称，可适当增加边缘 Padding 以营造呼吸感。

## 列表与网格卡片

| 模式 | 适用场景 |
|------|---------|
| 列表 | 信息较多、需要纵向阅读（学生列表） |
| 网格 | 信息较少、需要并列对比（功能入口） |

**规范**：

- 同一列表中字段顺序、位置、含义保持一致
- 区分"整卡片可点击"和"仅局部可点击"
