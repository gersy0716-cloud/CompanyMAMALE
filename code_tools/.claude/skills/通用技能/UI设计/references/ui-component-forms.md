# 表单设计规范

表单是用户输入的核心。最小化认知负担和输入错误是设计的关键。

## 🏗️ 结构与扫描

- **垂直列排布 (Vertical Column)**：标签和输入框应保持单列垂直排列。这符合人类垂直扫描的自然习性，比多列网格表单完成速度更快。
- **标签持久化 (Persistent Labels)**：
  - **禁止**：将 Placeholder 作为主要标签使用。
  - **原因**：一旦输入开始，Placeholder 消失，用户将失去上下文。始终使用可见的独立 Label。
- **必填项标注**：如果大多数项为必填，仅标注“（可选）”，减少视觉噪音。

## 🔘 交互逻辑

- **单选与下拉 (Selectors)**：
  - **<= 3 项**：使用 **Radio Buttons**。允许用户一眼看清所有选项进行对比。
  - **> 3 项**：使用 **Dropdown**。节省纵向空间，适用于较长的选项列表。
- **行动导向 CTA (Action-oriented)**：
  - 避免通用词汇如“提交”或“确定”。
  - 使用价值驱动的文案，如“创建免费账号”或“立即开始保存时间”。

## ⚡ 错误与反馈

- **行内校验 (Inline Validation)**：在用户离开输入框时立即反馈，而非在提交后。
- **输入提示 (Input Guides)**：Placeholder 应提供具体的格式示范（如 "<example@domain.com>"），作为用户的辅助引导。
