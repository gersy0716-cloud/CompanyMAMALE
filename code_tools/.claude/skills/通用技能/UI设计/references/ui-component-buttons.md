# 按钮组件规范

## 主要按钮 (Primary)

- **作用**：承载当前页面最重要的操作（提交、保存、生成）
- 每个独立视图只有一个主按钮
- 必须有文字标签，准确描述动作结果
- **Hero 区域准则**：顶部首屏区域只允许存在一个主 CTA 按钮，避免视觉竞争。
- **文案增强**：使用具体的动作（如“创建文章”）而非模糊的“提交”或“确定”。
- 完整状态集：默认、悬停、按下、禁用、加载中
- 键盘可通过回车触发

## 次要按钮 (Secondary)

- **作用**：辅助操作（返回、取消、更多设置）
- 视觉上不与主按钮争抢注意力
- 文案清晰表达辅助行为
- 同样需要禁用/加载状态

## 文本按钮 (Text / Ghost)

- **作用**：风险较低的快捷操作（查看详情、展开更多）
- 可嵌入段落或卡片内部
- 只用于点击区域较小、对结果不敏感的行为
