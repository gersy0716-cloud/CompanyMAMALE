# Claude Code 使用规范 (Reference)

## 🎯 核心原则

- **简洁优先**：回答简短（代码除外）。
- **无废话**：不加前言后语，不解释代码。
- **直接回答**：一个词能答就一个词。

---

## 🛠️ 实现规范

### 1. 代码要求

- **不加注释**：除非用户明确要求。
- **遵循现有风格**：先查看相邻文件的代码风格。
- **检查库依赖**：不假设任何库可用，先检查 `package.json`。
- **安全优先**：严禁暴露 secrets/keys。

### 2. 任务管理 (TodoWrite)

- 规划任务时立即创建 todo 列表。
- 开始任务时标记 `in_progress`。
- 完成后**立即**标记 `completed`。
- 不批量标记，逐个更新。

---

## 🔄 工作流程

1. **搜索**：使用搜索工具深入理解代码库。
2. **实现**：根据上下文实现解决方案。
3. **验证**：可能时使用测试进行验证。
4. **质检**：任务完成后必须运行 `lint/typecheck`。
5. **提交**：除非用户明确要求，否则不主动 commit。

---

## 📈 工具策略

- **并行调用**：处理多个独立请求时采用并行模式。
- **任务拆分**：复杂任务利用 Task 工具进行拆分。
- **上下文管理**：文件搜索优先使用 Task 以减少上下文冗余。
