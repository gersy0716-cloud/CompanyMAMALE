# 极致性能优化指南

## ⚡ 缓存层级策略

- **L1 短期 (内存)**：核心业务数据（学生/小组）缓存 10 分钟，数据变更时主动无效化。
- **L2 长期 (本地存储)**：不常变动的数据（节假日）缓存 24 小时。
- **L3 会话隔离 (Session)**：避免同一会话多标签页不必要的频繁同步。

## 🏗️ 加载与渲染优化

- **渐进式 UX**：同步渲染 Skeleton 或 Loading，挂载完成后异步填充数据。
- **并发请求**：利用 `Promise.all` 合并互不依赖的 API 请求，缩短总加载时间。

## 📊 DOM 操作实战

- **✅ 强烈推荐**：使用 `DocumentFragment`。
- **原理**：将多次 DOM 写操作（Reflow/Repaint）合并为一次，渲染成百上千个节点时性能提升可达 80% 以上。

## 🧮 算法与逻辑坑点

- **查找效率**：频繁的 ID 匹配应使用 `Set` 或 `Map` (O(1))，严禁在循环内套 `Array.find` (O(n²))。
- **日期偏移**：`toISOString()` 会输出零时区时间。业务上应使用自定义格式化函数：

  ```javascript
  const formatLocalMD = (d) => `${d.getMonth()+1}-${d.getDate()}`;
  ```
