# URL 参数处理与权限机制

## 🔐 敏感参数严禁默认填写

**禁止**在脚手架或文档示例中直接填写涉及具体租户和用户的参数。这些值必须动态从运行环境捕获。

| 参数名 | 业务含义 | 格式规范 |
|:---|:---|:---|
| `type` | 租户 Namespace | 如 `jj4x-api` |
| `token` | 鉴权令牌 | 标准 JWT |
| `userid` | 用户全局标识 | UUID |

## 🕹️ 核心业务模式判断

系统通过 URL 中的 `teachertoken` 存在与否来切换角色逻辑：

- **存在**：教师管理模式。
- **缺失**：学生展示模式。

## 🎓 学生端 API 调用细节

- **接口**：`/api/app/student/studentInfo`
- **班级字段坑**：班级信息存储在 `classs` 字段中（注意是 **3个 "s"**）。
- **取值逻辑**：`studentInfo.classs[0].id`
