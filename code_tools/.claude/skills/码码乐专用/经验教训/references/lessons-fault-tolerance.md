# 容错与防御性设计

## 🔄 数据一致性防御

常见场景：外部 API 已删除的学生在本地数据库 `student_ids` 字段中仍残留，导致渲染崩溃。

- **策略**：在通过 ID 列表渲染前，必须与最新的全量缓存（如 `APP_STATE.cachedStudents`）进行交叉过滤。

## 🧱 界面稳定性 (Fallback UI)

- **空状态占位**：即使后端返回空数组，DOM 结构也应保留最小框架（如 Tab 头），并展示“暂无数据”提示，避免页面完全空白或布局塌陷。
- **断网恢复**：关键静态数据加载失败时，优先尝试从 `localStorage` 的旧缓存中恢复展示。

## 📁 文件上传特异性

- **接口**：`POST /api/fileResouceItem/uploadUnified`
- **已知限制**：该接口方案不支持前端直接请求物理删除文件。
- **解决**：业务逻辑上仅删除数据库中的关联记录实现“逻辑删除”。
