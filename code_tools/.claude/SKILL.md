## ⚠️ Skill文档规范

### 核心原则

1. **简洁至上** — Context窗口是公共资源。只写Claude不知道的内容，每段文字都要值得它的token开销
2. **渐进式披露** — 三级加载：元数据(~100词,常驻) → SKILL.md本体(<500行,触发后加载) → 子模块(按需加载)
3. **适度自由** — 脆弱操作用脚本锁定，开放场景用文字指引

### SKILL.md结构

```
skill-name/
├── SKILL.md (必须)
│   ├── YAML frontmatter (必须: name + description)
│   └── Markdown正文 (必须, <500行)
└── 可选资源/
    ├── scripts/      - 可执行脚本(确定性任务)
    ├── references/   - 参考文档(按需加载到上下文)
    └── assets/       - 输出资源(模板/图片/字体,不加载到上下文)
```

### 关键规则

> [!IMPORTANT]
> **每个SKILL.md必须**：
>
> - 包含YAML frontmatter（`name` + `description`）
> - `description` 包含触发条件（决定何时激活此技能）
> - 正文控制在 **500行** 以内
> - 超出部分拆分到 `references/` 子目录
> - **不要创建** README.md / CHANGELOG.md 等辅助文件

### 内容拆分策略

> [!TIP]
> **何时拆分到 references/**：
>
> - 详细API文档、数据库Schema → `references/schema.md`
> - 多框架/多变体参考 → `references/aws.md`, `references/gcp.md`
> - 大量示例代码 → `references/examples.md`
>
> 拆分后必须在 SKILL.md 中标注引用路径和加载时机。
> 大文件(>100行)在顶部加目录，方便Claude预览全貌。

详细规范见: [技能开发](./skills/通用技能/技能开发/SKILL.md)
