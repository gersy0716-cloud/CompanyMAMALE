# 编码风格与质量规范

> 代码质量强制要求，包括注释规范、异常处理、分步迭代

---

## 通用规范

- 新功能必须包含单元测试
- 使用专业中文字体（微软雅黑、苹方、思源黑体）

---

## 渐进式开发流程

**核心理念**：框架先行，分块实现，逐步验证

### 开发步骤

1. **第一步：搭建项目框架**
   - 创建完整的文件目录结构
   - 编写基础的 HTML 骨架
   - 配置全局样式和状态管理
   - 封装 API 调用层
   - **验证点**：确保框架可运行，页面能正常加载

2. **第二步：分块实现功能**
   - 按功能模块逐个开发
   - 每完成一个模块后**必须停下来**，请求用户确认
   - **禁止**一次性实现所有功能

3. **第三步：迭代优化**
   - 优先验证核心业务场景
   - 核心功能稳定后再补充边缘情况

## 代码质量强制要求

### 1. 注释 + 逻辑解释（必须）

**禁止**只写代码不写注释，每个函数/模块必须包含：

```javascript
/**
 * 功能说明：从数据库获取学生列表并缓存
 *
 * 业务逻辑：
 * 1. 先检查缓存是否有效（10分钟内）
 * 2. 缓存失效则调用 API 获取最新数据
 * 3. 过滤掉已删除的学生
 * 4. 更新缓存和缓存时间戳
 *
 * @param {string} classId - 班级ID
 * @returns {Promise<Array>} 学生列表
 * @throws {Error} API调用失败时抛出错误
 */
```

**注释规范**：

- 函数头部：功能说明 + 业务逻辑步骤 + 参数说明 + 返回值 + 异常说明
- 关键代码块：解释"为什么这样做"而非"做了什么"

### 2. 异常处理（强制）

**所有**可能失败的操作必须包含异常处理：

```javascript
async function saveData(data) {
    try {
        // 1. 参数验证
        if (!data || !data.id) {
            throw new Error('数据格式错误：缺少必需字段 id');
        }

        // 2. API调用
        const response = await fetch('/api/save', {
            method: 'POST',
            body: JSON.stringify(data)
        });

        // 3. HTTP状态检查
        if (!response.ok) {
            throw new Error(`保存失败 (${response.status})`);
        }

        return await response.json();

    } catch (error) {
        console.error('❌ 保存数据失败:', error);
        alert(`保存失败: ${error.message}`);
        throw error;
    }
}
```

**必须处理的异常场景**：

| 场景 | 必须处理 |
|:---|:---|
| 网络请求 | 超时、404、500、网络断开 |
| 数据解析 | JSON 格式错误、字段缺失 |
| 用户输入 | 空值、格式错误、越界 |
| 第三方API | 调用失败、返回错误码 |

### 3. 分步迭代（强制）

**禁止**一次性生成完整代码，必须分步实现：

1. **核心场景**：只实现最基本的功能
2. **补充验证**：添加参数验证和格式检查
3. **完善细节**：添加异常处理和用户提示
