Execute the PRP file at $ARGUMENTS to implement the feature.

## 工作流程

### 1️⃣ 加载上下文

- 完整读取 PRP 文件
- 理解功能目标和约束
- 识别所有依赖文档

### 2️⃣ 创建计划

- 根据 PRP 的实现步骤创建详细任务列表
- 识别可并行执行的任务
- 标记关键验证点

### 3️⃣ 逐步执行

对每个步骤：

1. 实现代码变更
2. 运行验证门（测试/lint）
3. 修复发现的问题
4. 确认步骤完成后继续

### 4️⃣ 验证循环

```
┌─────────────────────────────────────┐
│  实现 → 测试 → 发现问题 → 修复     │
│         ↑                    │     │
│         └────────────────────┘     │
│              (循环直到通过)         │
└─────────────────────────────────────┘
```

### 5️⃣ 完成检查

确保所有成功标准满足：

- [ ] 所有验证门通过
- [ ] 代码符合项目规范
- [ ] 测试覆盖完整
- [ ] 文档已更新

## 执行规则

1. **不跳过验证门** - 每个步骤的验证必须通过才继续
2. **问题即修复** - 发现问题立即在当前步骤解决
3. **保持上下文** - 始终参考 PRP 中的约束和模式
4. **记录变更** - 完成后更新 `TASK.md`

## 输出

- 功能代码已实现
- 测试已通过
- 文档已更新
- `TASK.md` 已标记完成

---

> 💡 **使用方式**: `/execute-prp internal/PRPs/功能名称.md`
