Read the feature request file at $ARGUMENTS and create a comprehensive PRP (Product Requirements Prompt).

## 工作流程

### 1️⃣ 研究阶段

- 读取功能请求文件
- 分析代码库现有模式和约定
- 搜索相似实现作为参考
- 识别需遵循的架构模式

### 2️⃣ 文档收集

- 获取相关 API 文档
- 收集库/框架文档
- 记录已知问题和技巧

### 3️⃣ 蓝图创建

根据研究结果生成 PRP 文件，包含：

```markdown
# [功能名称] PRP

## 概述
[功能简述和目标]

## 上下文
- 相关现有代码引用
- 依赖的技能/规范文档
- API/库文档链接

## 实现步骤

### 步骤 1: [描述]
- 具体任务
- 验证门: [测试命令/检查点]

### 步骤 2: [描述]
...

## 测试要求
- [ ] 单元测试覆盖
- [ ] 边界情况测试
- [ ] 集成测试

## 成功标准
- [ ] 所有测试通过
- [ ] 代码符合项目规范
- [ ] 文档已更新
```

### 4️⃣ 质量检查

- 评估置信度（1-10 分）
- 确保所有必要上下文已包含
- 若置信度 < 7，标注需要澄清的点

## 输出

生成文件保存至: `internal/PRPs/[功能名称].md`

---

> 💡 **使用方式**: `/generate-prp docs/INITIAL.md`
