# 🏁 UI-UX-Pro-Max 发布前自检清单

在提交任何 UI 变更前，请对照此清单，确保达到 Pro Max 级别。

## 1. 物理层 (Physical)

- [ ] **行宽自检**：正文文字在桌面端是否因过长而发散？（控制在 700px 内）
- [ ] **圆角对齐**：嵌套元素的 `Outer_R` 是否等于 `Inner_R * 2`？
- [ ] **间距韵律**：是否严格遵循 4px 步进？组间间距是否明显大于组内间距？
- [ ] **呼吸感**：是否移除了不必要的分割线，改用 Whitespace 区分功能块？

## 2. 交互层 (Interaction)

- [ ] **Hero CTA**：视线首落点是否只有一个主按钮？
- [ ] **确切文案**：按钮是否写着“去哪里”或“做什么”（具体动作），而不是“提交”？
- [ ] **危险隔离**：删除等不可逆行为是否为红色，并有具体的对象确认（如“删除此课程”）？
- [ ] **反馈即时**：点击按钮或切换状态时，是否有哪怕 50ms 的过渡或 Loading 状态？

## 3. 认知层 (Cognitive)

- [ ] **价值预览**：关键行为旁是否有微型背书（如“1k+ 使用”、“无需信用卡”）？
- [ ] **视线偏置**：若使用了人像，其视线是否指向内容或 CTA 区域？
- [ ] **高亮偏置**：在多项选择中，是否通过 UI 引导了“最受欢迎”或“推荐项”？
- [ ] **空状态引导**：第一次进入该功能的用户，是否能通过“示例”或“模板”快速上手？

## 4. 移动端层 (Mobile)

- [ ] **Tooltip 替换**：所有的 Hover 交互是否已转换为点击/Tap 触发？
- [ ] **自适应布局**：是否为单列布局，且交互目标（按钮）尺寸不小于 44x44px？
