## 🎨 AI 漫画工厂集成踩坑 (必看)

### CORS 跨域与自定义 Header (重点)

 > [!WARNING]
 > 当 API 不支持 `Access-Control-Allow-Headers` 中的自定义头（如 `__tenant`）时，预检请求 (OPTIONS) 会失败。

- **表现**：报错 `Request header field __tenant is not allowed by Access-Control-Allow-Headers in preflight response`。
- **解决**：将 `__tenant` 从 Header 移至 **URL 查询参数** 中。

 ```typescript
 // ✅ 正确：绕过 Preflight
 const res = await fetch(`${apiUrl}?__tenant=${token}`, { ... });
 ```

### Iframe 无缝嵌入策略 (Iframe Embedding)

> [!TIP]
> **最佳实践**：不要强行把 Next.js 项目塞进现有轻量级项目，直接采用 `iframe` 嵌入正在运行的 Next.js 服务，保持原始项目原汁原味且零侵入。

- **跨域通信 (Navigation)**：在 iframe 内部通过 `window.parent.postMessage({ type: 'navigate-home' }, '*')` 向外传递返回主页的指令，父窗口监听该消息并执行路由跳转 `navigate('/')`，完美解决跨端口跨域问题。
- **UI 对齐 (Visual Alignment)**：在嵌入源 (ai-comic-factory) 的 `main.tsx` 中直接补充“返回主页”按钮，使其与历史记录页面的按钮在绝对位置上完全一致（`top-8 left-12`），并且去掉外层的返回按钮。
- **防止遮挡**：在主应用的 `hero-section` 增加 `pt-32` 内边距，使主体内容整体下移，避免与顶部绝对定位的返回按钮发生视觉冲突。

### 历史记录同步与补全 (Guard Logic)

- **ID 大小写不一**：BaseMulti 返回的历史记录中 ID 字段可能是 `id` 或 `Id`。加载和同步时必须兼容：`const dbRecordId = record.id || record.Id`。
- **差异化保存**：在浏览历史记录时，严禁全量重写记录。应通过逻辑判断，仅在检测到有新生成的图片（缺失补全）时，调用 `updateComicRecord` 补全 `panel_images`。
- **旧画风 ID 兼容 (Fallback)**：由于业务迭代可能删除或重命名某项画风预设(如拆分了原本的 `children_book`)，在加载历史画风时必须增加安全回退：`preset: getPreset(record.style) || getPreset("children_book_watercolor")`。同时，默认初始状态已改为 `random` (随机风格)。
- **状态联动**：只有当所有分镜图片均已生成且合规时，才将 `status` 更新为 `已完成`。
- **渲染卡死规避 (Critical)**：漫画生成逻辑必须包裹在 `try...finally` 中，确保无论成功与否，都要重置 `isGeneratingStory` 状态，彻底杜绝加载动画（Loading Modal）永久卡死。
- **渐进式渲染 (UX)**：在生成过程中，应采用增量更新（Incremental Update）而非全量覆盖 store，在分镜数据到达时立即渲染占位符，提升用户感知的交互速度。

### 🎙️ 语音识别集成 (ASR Hybrid Version)

> [!TIP]
> **最佳平衡方案**：同时集成浏览器原生 API 和后端大模型 API 兜底。

- **逻辑流**：优先检测 `window.SpeechRecognition`（免费、实时、无限量）。若不支持（如在第三方内置浏览器、受限内嵌 App），则自动降级到 `MediaRecorder` 手动录音。
- **录音转换**：`MediaRecorder` 生成的 Blob 需通过 `FileReader` 转为 Base64 格式，并剥离 DataURL 前缀（若 API 要求纯 Base64）。
- **认证共用**：语音接口（如 `api/app/volcengine/audioToTextMy`）应使用与绘图/提示词接口一致的 `token` 和 `teachertoken` 认证体系。
- **交互细节**：识别过程中应显示独立的“正在识别...”加载态，并设定超时/权限拒绝（NotAllowedError）的友好的报错挽留对话框。

### UI 审美与空间优化 (Visual Refinement)

- **字体比例**：分镜中的旁白（Caption）字体应根据 `zoomLevel` 动态缩放，且默认应比对话气泡（Speech）更小、更淡，避免喧宾夺主。
- **首屏压缩**：Hero 区域的 `padding-top` 应保持在极小范围（如 `pt-2` 或 `pt-4`），并移除冗余间距，将用户核心注意力上移至输入框。
- **沉浸式视口**：漫画容器应使用 `vh`（如 `min-h-[92vh]`）限制，确保在大屏幕和笔记本上都能一眼看到全貌而无需频繁下滚。
- **去背景化**：移除漫画底部的“毛玻璃面板”（Glass Card），采用无背景、无投影的“浮动式”布局，使漫画本身成为唯一的视觉中心。

- **减法设计**：移除历史记录卡片上干扰视觉的“播放按钮”覆盖层和“镜数（Layers）”统计，保持书架感的纯净度。

### API 并发与请求排序 (Concurrency Control)

> [!IMPORTANT]
> 当调用 Jimeng 或类似图片生成接口时，**严禁并行请求**！
> 该接口通常有严格的单用户/单租户并发限制 (Concurrent Limit)。

- **排队机制**：在 `render.ts` 中实现全局 Promise 链或队列。
- **缓冲时间**：请求间隔建议增加 **1000ms** 的安全缓冲。
- **错误捕获**：当捕捉到 `Concurrent Limit` 错误时，前端应展示友好的重试引导（如“绘图服务器正忙”），并设定异步自动重试。

### 儿童/绘本 AI 画风规避逻辑 (Prompt Engineering)

> [!WARNING]
> 生成“儿童插画”或“婴儿”主题时，极易触发 **Uncanny Valley (恐怖谷效应)** 或生成肢体畸形的恐怖图片（如带猪耳的婴儿、惊悚的写实狐狸、人类肢体组合）。

- **画风细分控制**：切忌使用宽泛的 `children's book`，应拆分为具体的可控画风，如 `watercolor and gouache style` (柔和手绘) 或 `bright flat colors, kawaii style` (明亮平涂)。
- **强制非写实与降维锚点**：Prompt 中必须包含强烈锚点，如 `flat vector illustration`, `naive art`, `storybook aesthetic`。若涉及动物，必须抛弃 `anthropomorphic` (拟人化) 等容易产生写实毛发的词，改用简单二维卡通锚点如 `Miffy style`, `Animal Crossing style`, `Peppa Pig style`。
- **严禁写实反向消除**：Negative Prompt 核心词：`photographic`, `realistic`, `3D render`, `uncanny valley`, `human skin texture`, `hyper-realistic`。对于特定场景（如动物），强行排除 `furry art`, `realistic pig/wolf`, `human body`, `human hands on animals`, `creepy faces`。
- **照明限制**：禁止 3D 光影（`no 3D lighting`，负向屏蔽 `harsh shadows`），强制使用扁平化或柔和化（`soft magical lighting`）色彩。

## 🗄️ BaseMulti 数据库踩坑

### 已知限制

- ❌ **filter/sort 参数不生效**：需在客户端处理
- ❌ **PATCH 被 CORS 阻止**：需安装 Allow CORS 浏览器扩展并配置白名单 `data.520ai.cc`
- ❌ **PUT 返回 404**：BaseMulti 不支持 PUT

### 字段类型

- **Number 字段必须传数字**，字符串会导致 500 错误

```javascript
// ✅ 正确
const userId = parseInt(Auth.currentUser.id);

// ❌ 错误 — "1" 而非 1
d.uploader_id = Auth.currentUser.id;
```

### 方法名

- 删除记录：`DB.remove()` 而非 `DB.delete()`
- 可用方法：`query`, `getById`, `create`, `update`, `remove`, `queryAll`, `uploadFile`

### 建表规范

- Single Select 必须手动添加所有 Options 的 Value
- 建议开启 `Value = Label`

### 发布新版本 500 错误 (重点)

> [!IMPORTANT]
> 向 `VERSIONS` 表 (`3QBYQzM3Hs`) 创建记录时：
>
> 1. **严禁**携带 `user_id` 字段！系统表不支持自定义业务映射字段，携带会导致 500 服务器错误。
> 2. **必须**提供 `name` 字段，且建议设置为版本号（如 `v1.0.2`）。

```javascript
// ✅ 正确：最小化有效载荷
await DB.create(Config.TABLES.VERSIONS, {
    name: 'v1.0.1',
    version: 'v1.0.1',
    download_url: '...',
    changelog: ''
});
```

## 🖥️ 公司管理系统速查

### 当前版本

- `Config.CURRENT_VERSION`：`v1.0.0`（每次发版需更新）

### 权限

- **版本发布**：仅 `admin` 角色或 `admin` 用户可上传新版本。
- **消息过滤**：自己发出的消息不在自己的列表中

### 通知系统

- 仅使用 Windows 原生通知（Electron `Notification` API）
- 轮询间隔：60 秒（`NOTIFICATION_POLLING_INTERVAL`）

### Preload 桥接 API

| API | 用途 |
| :--- | :--- |
| `showNotification(title, body)` | 系统通知 |
| `updateBadge(count)` | 托盘角标 |
| `onNavigateTo(callback)` | 导航跳转 |
| `speakText(text)` | TTS 语音 |
| `openExternal(url)` | 打开外部链接 |
| `downloadAndInstallUpdate(url)` | 触发静默下载并安装 |
| `onUpdateProgress(callback)` | 监听下载进度更新 UI |
| `onUpdateError(callback)` | 监听下载错误 |

### Modal API

- 使用 `Modal.open(title, bodyHTML, footerHTML)` 而非 `Modal.show()`
- `footerHTML` 是字符串格式的 HTML 按钮，非数组对象

## 🚀 版本管理与自动更新流程

### 1. 本地打包

- 修改 `js/config.js` 中的 `CURRENT_VERSION`（如从 `v1.0.0` 改为 `v1.0.1`）。
- 修改 `package.json` 中的 `version` 字段。
- 运行 `npm run build` 生成 `码码乐 Setup 1.x.x.exe`。

### 2. 云端托管 (七牛云)

- 将生成的 `setup.exe` 手动上传到七牛云公开空间。
- 复制该文件的 **下载直链**（确保粘贴到浏览器能直接弹出下载）。

### 3. 发布版本

- 登录账号 `luowenbin`。
- 进入「版本更新」页面，点击「发布新版本」。
- 输入新版本号、更新说明和复制的**七牛云直链**。

### 4. 自动分发

- 系统每 30 分钟后台轮询一次数据库。
- 发现新版后，所有在线用户会自动开始后台静默下载。
- 下载完成后，程序静默退出、覆盖安装并自动重启，用户全程无感。
