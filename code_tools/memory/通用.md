## 🛠️ 环境踩坑

### PowerShell 中文路径

**问题**：PowerShell 对中文字符路径编码异常，导致"路径不存在"错误。

**解决**：

- 使用通配符匹配：`Get-ChildItem "f:\工作\码码乐\工作*开发\..."`
- 创建 `.bat` 脚本启动程序
- 使用 `Start-Process` 配合通配符匹配结果

### Electron 启动

- IDE 终端会设置 `ELECTRON_RUN_AS_NODE=1`，导致 `require('electron')` 返回路径字符串
- **解决**：通过 `start.bat` 启动，或先执行 `Remove-Item Env:ELECTRON_RUN_AS_NODE -ErrorAction SilentlyContinue`

### Electron 调试与生产配置

- **DevTools 控制**：在 `webPreferences` 中设置 `devTools: true/false`。
- **强制打开**：`mainWindow.webContents.openDevTools()` 常用于开发环境（`IS_DEV` 模式）。
- **生产环境**：打包前务必关闭 `devTools` 并禁用调试代码，推荐使用全局 `Config.IS_DEV` 统一管理。

### Electron 打包 (Windows)

**问题**：`electron-builder` 报 `Cannot create symbolic link` 或 `exit status 2`

**原因**：符号链接权限 + 中文路径编码

**解决**：

1. 项目移到纯英文路径（如 `C:\mamale-desktop`）
2. 设置 `ELECTRON_BUILDER_CACHE="C:\eb-cache"` 避开中文
3. 或改用 `electron-packager`（`--asar --dir`）

### Windows 文件占用

**问题**：`rd /s /q` 报"拒绝访问"

**解决**：`taskkill /f /im "程序名.exe" /t` 然后重试

### 浏览器缓存

**问题**：修改 JS 后浏览器仍执行旧代码

**解决**：`Ctrl + Shift + R` 或 `Ctrl + F5` 强制刷新

## 🐛 代码踩坑

### 脆弱的 DOM 选择器

**问题**：`.form-group:nth-child(4)` 等索引选择器在 UI 微调后失效。

**解决**：优先使用唯一 `id` 或语义化 `data-attribute`。

### 语法嵌套陷阱

**问题**：复杂 `try-catch` 块中添加逻辑时闭合错误括号，导致整个应用无法加载。

**解决**：大规模修改后检查逻辑块完整性。

### 数据库同步冲突

**问题**：本地缓存覆盖云端最新数据，用户上传的文件"消失"。

**解决**：严格"云端优先"，仅在云端缺失时用本地缓存。

### 模态框圆角

**问题**：设了 `border-radius` 但角是尖的。子元素内容溢出遮挡圆角。

**解决**：给容器加 `overflow: hidden`。

## 📐 设计模式参考

### 个性化数据存储

**场景**：同一实体需存储不同用户的个性化设置

**方案**：JSON 字段存 `{ "userId": "value" }` 映射

```javascript
// 读取
const userTypes = JSON.parse(task.user_task_types || '{}');
return userTypes[String(Auth.currentUser.id)] || '默认值';

// 更新
userTypes[String(Auth.currentUser.id)] = newType;
await DB.update(table, id, { field: JSON.stringify(userTypes) });
```

**注意**：用户 ID 必须转为字符串作为键

### 视图上下文传递

**场景**：同一组件在不同上下文中显示不同内容

```javascript
const Component = {
    currentContext: 'global', // 'global' or 'personal'
    show(id, context = 'global') {
        this.currentContext = context;
        // 根据上下文渲染
    }
};
```
