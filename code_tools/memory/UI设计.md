# UI/UX 设计偏好积累 (Memory)

> 📌 **定位**：记录在开发过程中逐渐积累的审美偏好与交互逻辑，作为后续 UI 实现的参考依据。
>
> **何时读取**：执行前端开发、UI 设计或交互优化任务前。
>
> **核心逻辑**：审美随开发过程动态积累，本文件非死板规则，而是 AI 学习用户偏好的进化日志。

## ✨ 视觉审美倾向

### 1. 核心风格：清透果冻系

- ✅ **呼吸感**：低饱和度 + 柔光感，让应用显得精致且不压抑。
- ✅ **玻璃拟态 (Glassmorphism)**：大量使用半透明 (`rgba`) + `backdrop-filter: blur()`。
- ✅ **环境色阴影**：阴影取自定义主色的低透明度版本，如 `rgba(主色, 0.12)`，严禁使用纯黑/生硬阴影。
- ✅ **微发光**：在容器边缘使用淡淡的白色或主色内发光。

### 2. 形态与布局

- ✅ **大圆角**：卡片层级建议 16-20px，按钮 12-16px，小标签 10-12px。
- ✅ **2x 圆角法则**：外层容器圆角 = 内层元素圆角 + Padding 间距。
- ✅ **Whitespace 优先**：优先使用较大的负空间 (Margin/Padding) 来区分区块，而非使用实线分割。
- ✅ **渐变美学**：主按钮使用同色系微弱渐变，配合发光和环境色阴影。

### 3. 禁止项 (Anti-Patterns)

- ❌ **冷淡 B 端风**：避免 Arco/AntD 那种极简、扁平、灰白主调的传统企业后台感。
- ❌ **硬边缘**：避免生硬的 1px 黑线边框。
- ❌ **纯色大块**：避免没有细节的大面积高饱和度色块。

## 📐 布局与组件模式 (Patterns)

### 1. Bento Box 布局 (Bento Layout)

- **规格**：核心工作区采用模块化网格，大屏建议每行 3-4 个卡片。
- **细节**：卡片间距保持一致 (Gap: 16-24px)，通过不同权重的卡片尺寸实现视觉层次感。

### 2. Tab Bar 导航规范 (Mobile/Sidebar)

- **图标风格**：极简线条艺术 (Minimalist Line Art)。
- **色彩**：默认状态使用低饱和灰色/环境色，活跃态推荐使用 **鼠尾草绿 (Sage Green)** 或品牌主色。
- **规格**：图标标准尺寸建议为 81x81px (针对高分屏优化)。

### 3. 卡片化信息呈现 (Card-First)

- **原则**：减少二级弹窗，优先在卡片内平铺展示核心信息。
- **示例**：账号管理中，直接在卡片内显示账号/密码（支持一键复制），而非跳转详情页。
- **交互**：卡片右上角放置“编辑”等次级操作，保持主体区洁净。

### 4. 实验场模式 (Playground)

- **布局**：采用“左侧导航 + 右侧内容”的类文档结构。
- **演示卡**：包含“实时预览 + 标题描述 + 代码复制 + 展开源码”四要素。

---

### 5. 细节处理 (UX Details)

- **空状态表现**：列表或搜索为空时，提示信息与图标必须水平+垂直**居中**，避免页边距倾斜。
- **卡片头部操作**：操作按钮（编辑、删除）优先放置在卡片 Header 的右侧。
- **列表视觉**：多用户发布的内容，应在卡片底部或角落清晰标注发布者姓名。

## 🖱️ 交互逻辑偏好

### 1. 按钮与操作

- **位置锚定**：在复杂详情页或侧边栏中，核心操作按钮（如“保存”、“提交”）建议固定在容器**右下角**，符合用户习惯。
- **冗余消除**：移除重复的交互按钮，增强核心区域的交互深度。

### 2. 交互深度

- **拖拽区增强**：支持点击交互。例如：左键点击执行文件操作，右键点击执行文件夹/高级操作，配合明显的悬浮态提示。
